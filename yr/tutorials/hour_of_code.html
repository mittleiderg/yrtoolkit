Hour of Code

Welcome to the MIT App Inventor Hour of Code!

Are you ready to build your first mobile app with MIT App Inventor? Make sure you followed the instructions from the Connect tab to live-test your app as you build it!

In this hour, we will build an app that plays a buzzing sound when you click on a picture of Codi the bee.

Introduction to Environment

There are two main components to the App Inventor environment: the Designer and the Blocks Editor.

The Designer allows you to add components to your app and lay out what it will look like.

The Blocks Editor allows you to code your app to give it functionality--that is, make it do stuff!

You can toggle between the two environments via the buttons in the top right corner.

Designer Components

Within the Designer, there are five windows: Palette, Viewer, Components, Media, and Properties.

The Palette is where all available components are. You can drag and drop any component from the Palette into the Viewer.

The Viewer is where you can see what the app will look like.

The Components menu shows all available components that have been added to the app. Note that the Screen component is always available as a component.

Media shows any uploaded files. There should be three files already pre-uploaded. Note: If you do not have these files, you can download them here (and use the Upload File button to upload them into your app):

  • Icon
  • Codi
  • Buzz Sound

    • Finally, the Properties window allows you to manipulate any of the finer details of the currently selected component.

    Add Button

    From the User Interface palette, drag in a Button, and drop it onto the Viewer.

    Add Sound

    From the Media palette, drag in a Sound component, and drop it onto the Viewer.

    Because this is a non-visible component, the Sound will drop to the bottom, rather than appearing on the Viewer screen itself.

    Button Properties

    In the Components window, click on Button1. The Properties window for Button1 will appear.

    Set the Height property to Automatic and the Width property to Fill Parent. This will allow Codi to be a large square the width of your device's screen.

    Set the Image property to codi.jpg.

    Set the Text property to be empty.

    If you are connected to the Companion app, you should now see a mirroring Codi on your device screen!

    Sound Properties

    Under Source, select Bee-Sound.mp3.

    Let's Write Some Code!

    Switch to the Blocks Editor.

    You'll see that there are three windows in the Blocks Editor - Blocks, Media, and Viewer.

    Media is exactly the same as in the Designer.

    Blocks has three categories: built-in blocks, component blocks, and generic blocks.

    Notice that the component blocks are exactly the same as what is in the Components window from the Designer.

    Add Button Click Event

    From the Button1 drawer, drag in a when Button1.Click event block, and drop it onto the Viewer.

    Add Sound

    From the Sound1 drawer, drag in a call Sound1.Play block, and snap it into the Button Click block.

    Also drag in a call Sound1.Vibrate block. Snap it in underneath the Play block.

    Add Built-In Block

    From the Math drawer, drag in a 0 block.

    Snap it into the millisecs socket of the Vibrate block. Here, millisecs is a parameter: It needs to be told how long to vibrate for.

    Change the 0 to 50.

    Test your App!

    What happens when you click the picture of Codi in the Companion app? Does it play a sound and vibrate the device?

    If the vibration does not work, your device may not support this function.

    Congratulations! You've built your first app. You're an App Inventor!

    Install App

    If you would like to install your app onto your device, navigate to the Build menu at the top of the screen, and click App (provide QR code for .apk). This will build your app and provide you a scannable QR code that will let you install your app onto your device!
